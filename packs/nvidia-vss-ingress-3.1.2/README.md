# NVIDIA VSS Ingress (envoy + sdr + vios-ingress + proxy)

NVIDIA VSS 3.x ingress / routing layer — envoy-proxy (xDS stream router), sdr (service discovery & routing control plane), vss-vios-ingress (nginx VST data-plane front door :30888) and vss-proxy (single public front door :7777, no_proxy-by-default). All services are multiarch 3.1.0 (no -sbsa variant). Manifest-only pack; references the ngc-pull-secret created by the data-infrastructure pack and the redis/sdr/streamprocessing/vss-agent/vss-ui/vss-vios-sensor Services from the other packs. install-priority 18. Part of the VSS 3.x five-pack set (data-infrastructure + nims + vllm + core + ingress); all five belong in every profile. KNOWN BLOCKER: sdr uses Docker-socket container discovery (WDM_CLUSTER_TYPE=docker) which is not portable to k8s and must be reworked for a kubernetes cluster mode before sdr/envoy routing functions.
